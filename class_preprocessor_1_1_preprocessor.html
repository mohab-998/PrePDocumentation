Detailed Description

Class: preprocessor.

Extracts cutouts of symbols via three color filters: Red, Green and Yellow.

Class works as a callable object, i.e. running the constructor will generate all symbol cutouts and export them to the specified, export directory.

Generates an Pascal VOC style XML file for symbol cutouts.

Bounding boxes are first mapped from the original plan's coordinate system to the new symbol cutouts.

Note
Since the color filters are not perfect, there is often a notable difference between the symbol's bounding box and the color filter's cutout and therefore setting the labels to [0, 0, width, height] is incorrect.
Warning
If no labels were found in the cutout, the JPG file is removed, otherwise the XML file would be empty and generate a run-time error.
Parameters
exportPath:strExport directory.
cutoutBBox:list<int>Bounding box of symbol cutout in the original plan's coordinate system.
symbolName:strName (class/label) of symbol.

private

Uses one mask to try and extract as many symbols as possible for a given color.

For "red" and "yellow", __isolateColor() and contours are extracted from the result.

Contours are only extracted as valid symbols if they have no children or parents (purely empirical/trial and error; can be changed).

Note
Function terminates for "green" since generally one mask is not enough for green (a lot of backgrounds in emergency floor plans are green).
Parameters
color:strFilter color (red, green or yellow). Will throw an error if any other color is used.
Exceptions
IllegalArgumentCannot have any other input argument than "red", "green" or "yellow".
Returns
counter:int - Number of extracted symbols.

)
static

Calculates the accuracy of the preprocessor (color filters).

The average percentage (across all classes) and the mean percentage per class are then exported to a *.txt file.

Note
This function only compares the number of identified symbols and compares it to the total number of symbols.
It ignores the location of bounding boxes, e.g. via IoU or other metrics.
Warning
Function will only work if the XML files exist in both directories, i.e. do not run after using Preprocessor.type = "test".
Parameters
originalPlanXMLPath:strPath to original plans' XML files.
preprocessedFolder:strDirectory containing the extracted symbols.
Returns
precentagePerClass:dict<str, double> - Accuracy percentage for each class.
meanPercentage:double - Average accuracy (percentage) for all classes.

def Preprocessor.Preprocessor.setupValidation ( bool   forTesting = False)
static

Sets up a Preprocessor object.

Initially all original plans are moved from the export directory.

Then for each original plan, a new Preprocessor object is set up and used to extract all symbols.

The symbols are then exported to the export directory.

Warning
Functions' name is misleading. In addition to the validation dataset, this function can be used to set up the testing dataset.
Parameters
forTesting:boolOPTIONAL - Set up the preprocessing objects for the validation dataset (False/default) or the testing dataset (True).
